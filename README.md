# The Grand Table — Restaurant Order & Billing Management System

A full-stack restaurant management app for taking orders, kitchen workflow, cashier billing, admin analytics, and printable receipts. Built with **React (Vite)**, **Node.js / Express**, **MongoDB**, and **Chart.js**.

---

## Features

### Role-Based Panels

| Role | What they can do |
|------|------------------|
| **Admin** | Dashboard, Analytics (charts), menu/categories/tables management, staff management, orders & bills history |
| **Waiter** | View table layout, take & place orders |
| **Kitchen** | View incoming orders, mark as in-kitchen / served |
| **Cashier** | Generate bills for served orders, apply discounts, accept payments (Cash / Card / UPI), print receipts |

### Core Functionality

- **Full CRUD** for menu items, categories, tables, and staff
- **Order lifecycle**: Waiter places → Kitchen processes → Cashier bills
- **Payment methods**: Cash, Card (Visa / Mastercard / RuPay with simulated processing), UPI (Google Pay / PhonePe / Paytm / BHIM with QR code)
- **Printable receipts** styled for 80mm thermal printers
- **Dark mode** toggle with glassmorphism UI
- **Responsive design** — works on mobile, tablet, and desktop
- **Role-based authentication** with JWT

### Admin Pages

| Page | URL | Description |
|------|-----|-------------|
| Dashboard | `/admin/dashboard` | Today's stats (orders, revenue, pending, tables) + recent orders table |
| Analytics | `/admin/analytics` | Revenue trend, busiest days, top items, payment method breakdown (Chart.js) |
| Menu | `/admin/menu` | Add / edit / delete / toggle availability of menu items |
| Categories | `/admin/categories` | Manage food categories |
| Tables | `/admin/tables` | Manage tables & view QR codes |
| Staff | `/admin/staff` | View all staff, add new staff members |
| Orders & Bills | `/admin/billing` | View all paid bills with date filter, view/print receipts |

## Setup

### 1. Backend

```bash
cd backend
npm install
```

Create environment file:

```bash
copy .env.example .env
```

On Mac/Linux:
```bash
cp .env.example .env
```

Edit `backend/.env`:

```env
PORT=5000
MONGODB_URI=mongodb://127.0.0.1:27017/restaurant_db
JWT_SECRET=your_super_secret_jwt_key_change_this
```

Load demo data (menu items, tables, staff accounts):

```bash
npm run seed
```

### 2. Frontend Dependencies

From the root directory:

```bash
npm install
```

## Running the Application

### Development (with hot reload)

Open **two terminals**:

**Terminal 1 — Backend API:**
```bash
cd backend
npm run dev
```

**Terminal 2 — Frontend (Vite dev server):**
```bash
npm run dev
```

Frontend: `http://localhost:5173`  
Backend API: `http://localhost:5000`

### Production (single command, serves everything)

```bash
npm run build
npm start
```

Opens at: `http://localhost:5000`

### Access from Other Devices (phone, tablet)

1. Run `npm run build` then `npm start`
2. Look for `Network: http://192.168.x.x:5000` in the terminal
3. Connect your phone/tablet to the **same WiFi**
4. Open Chrome and type that URL

> **Note:** MongoDB must be running on your PC for data to work.

## Payment Methods (Cashier Panel)

### Cash
Click **Cash** → **Confirm Cash Payment** → payment recorded.

### Card
1. Click **Card** → **Pay with Card**
2. Choose **Visa**, **Mastercard**, or **RuPay**
3. Watch simulated processing → auto-success → payment recorded

### UPI
1. Click **UPI** → **Pay with UPI**
2. Scan the QR code with any UPI app, or tap **Google Pay / PhonePe / Paytm / BHIM**
3. Watch simulated processing → auto-success → payment recorded

## Typical Workflow

1. **Waiter** — Log in → pick a table → add menu items → place order
2. **Kitchen** — Log in → open order → mark **In Kitchen** → mark **Served**
3. **Cashier** — Log in → select served order → generate bill → apply discount → choose payment method → confirm → print receipt
4. **Admin** — Dashboard for today's stats → Analytics for trends → Orders & Bills for history → Manage menu/categories/tables/staff

## Troubleshooting

### "Forbidden" or "Failed to load data"
- Log in as **Admin** with the **Admin** role selected
- Restart the backend after updates
- Clear browser localStorage and log in again

### "Failed to generate bill"
- Kitchen must mark the order **Served** first
- Log in as **Cashier** or **Admin** with the correct role

### Cannot see past bills
- Clear the date filter in **Orders & Bills** to show all dates
- Bills are never deleted — only filtered by date in the UI
- Ensure MongoDB is running and seed was not re-run

### Staff not showing in list
- Click **Refresh** on the Staff page
- Registered users appear in the same collection as admin-added staff

### Analytics shows no data
- Complete at least one payment in the Cashier panel
- Click **Refresh** on the Analytics page

### MongoDB connection error
- Start MongoDB locally, or fix `MONGODB_URI` in `backend/.env`
